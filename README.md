Proiect TAS - Testarea Automată a Sistemelor Software
Site vizat: Sweet Shop

📝 Descrierea Proiectului
Acest proiect are ca scop demonstrarea competențelor de asigurare a calității software (QA) prin aplicarea mai multor tipuri de testare pe o aplicație web de tip e-commerce. Proiectul acoperă întreg ciclul de viață al testării, de la planificare și proiectarea cazurilor de testare, până la execuție și raportarea bug-urilor.

🛠️ Tehnologii și Instrumente utilizate
Proiectul este structurat pe patru piloni principali de testare:

Testare Manuală: Utilizată pentru explorarea inițială a UI/UX și validarea fluxurilor simple.

Testare Funcțională Automatizată: * Limbaj: Java

Framework: Selenium WebDriver

Abordare: BDD (Behavior Driven Development) folosind Cucumber și Gherkin.

Testare de Performanță: * Instrument: Apache JMeter

Tipuri de teste: Load Testing și Stress Testing pentru verificarea stabilității serverului sub sarcină.

Testare de Securitate: * Instrument: OWASP ZAP

Scop: Identificarea vulnerabilităților de tip XSS, SQL Injection și lipsa headerelor de securitate.

📋 Structura Proiectului
Proiectul include:

Machetă de testare (Excel): Conține 13 cazuri de testare (5 Funcționale, 5 Performanță, 3 Securitate).

Documentație (Word/PDF): Descrierea detaliată a strategiei, execuției și a rezultatelor obținute (inclusiv dovezi/screenshots).

Soluția de automatizare: Codul sursă Java integrat cu Selenium și Cucumber.

Fișiere de configurare: Planuri de test .jmx (JMeter) și rapoarte de securitate (ZAP).

🚀 Obiective urmărite
Validarea funcționalităților critice (Login, Coș de cumpărături, Checkout).

Asigurarea unui timp de răspuns optim sub sarcina a 50+ utilizatori simultani.

Identificarea și raportarea oricăror breșe de securitate care ar putea expune datele utilizatorilor.
